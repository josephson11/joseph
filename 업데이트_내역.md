## 🛠️ 빌드 오류 수정 (2026-02-05)

### 🚨 문제 상황
`npm run build` 실행 시 두 가지 오류 발생:
1. **PowerShell 보안 정책**: 스크립트 실행 권한 문제로 `npm` 실행 불가
2. **코드 컴파일 오류**: 
   - `Unsupported Server Component type: undefined`: `Instagram` 아이콘 매핑 누락으로 인한 렌더링 실패
   - `Cannot find module 'critters'`: `optimizeCss` 설정 활성화로 인한 의존성 모듈 찾기 실패

### ✅ 해결 방법

#### 1. 코드 수정
| 파일 경로 | 수정 내용 |
|---------|---------|
| `next.config.js` | `optimizeCss: true` 설정 제거 (실험적 기능 비활성화로 안정성 확보) |
| `src/components/ui/Footer.tsx` | `lucide-react`에서 `Instagram` 아이콘 임포트 및 `iconMap`에 추가하여 매핑 오류 해결 |
| `src/components/sections/Contact.tsx` | `lucide-react`에서 `Instagram` 아이콘 임포트 및 `iconMap`에 추가하여 매핑 오류 해결 |

#### 2. 실행 환경 개선
PowerShell 보안 정책으로 인해 `npm run build`가 직접 실행되지 않는 문제를 해결하기 위해, `node`를 직접 실행하는 배치 파일(`build.bat`)을 생성했습니다.

### 🚀 빌드 방법
이제 다음 두 가지 방법 중 하나로 빌드가 가능합니다:

1. **(권장) `build.bat` 더블 클릭** 또는 터미널에서 실행:
   ```cmd
   .\build.bat
   ```
   *이 방법은 시스템 보안 정책과 무관하게 작동합니다.*

2. **PowerShell 정책 변경 후 `npm run build` 실행**:
   (관리자 권한으로 PowerShell 실행 필요)
   ```powershell
   Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
   npm run build
   ```
